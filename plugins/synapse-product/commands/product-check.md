# Product Check — 제품 원칙 검증

## 개요

새로운 기능이나 변경사항이 시냅스의 제품 원칙, UX 가이드라인, 기능 범위에 부합하는지 종합적으로 검증합니다.

## Interactive-First

**CRITICAL**: 인자가 없어도 실패하지 마세요. 대화형으로 안내합니다.

인자가 제공되지 않은 경우:
```
"어떤 기능이나 변경사항을 검증하고 싶으신가요?"
Options: "새 기능 기획", "기존 기능 변경", "버그 수정 영향 분석", "직접 설명"
```

## 워크플로우

### Phase 1: 기능 이해

사용자가 검증하려는 기능/변경사항을 정확히 이해합니다.

1. **기능 설명** 수집
   - 기능의 목적과 동작 방식
   - 영향을 받는 모듈/페이지
   - 타겟 사용자 역할

2. 추가 정보가 필요한 경우:
   ```
   "이 기능의 주요 사용자는 누구인가요?"
   Options: "데이터 관리자", "라벨링 작업자", "프로젝트 관리자", "ML 엔지니어", "모든 사용자"
   ```

### Phase 2: 제품 원칙 검증

5가지 핵심 원칙에 대해 순서대로 검증합니다.

#### 2.1 All-in-One MLOps 검증

- 기존 워크플로우와의 연결성 확인
- 외부 도구 의존성 확인
- End-to-End 파이프라인 내 위치 파악

#### 2.2 라벨링 최적화 검증

- 라벨링 효율성에 미치는 영향
- 어노테이터 워크플로우 복잡도 변화
- AI 보조 기능 활용 여부

#### 2.3 No/Low/High Code 검증

- 사용 난이도 수준 확인
- UI 접근성 확인
- SDK/API 확장 가능성 확인

#### 2.4 보안 검증

- 데이터 유출 위험
- RBAC 준수 여부
- 인증/인가 처리

#### 2.5 확장성 검증

- 플러그인 호환성
- API 하위 호환성
- 대량 데이터 처리 능력

### Phase 3: 범위 판단

기능 범위 판단 프레임워크를 적용합니다.

```
1. AI 학습 데이터 구축/관리와 관련이 있는가?
2. End-to-End 파이프라인의 어느 단계에 해당하는가?
3. 모든 사용자에게 필요한가, 특정 도메인에 한정되는가?
```

판단 결과:
- **In-Scope (플랫폼 내장)**: 모든 사용자에게 필요한 핵심 기능
- **플러그인**: 특정 도메인/사용 사례에 한정
- **Out-of-Scope**: 시냅스의 범위 밖

### Phase 4: 결과 보고

검증 결과를 구조화된 형태로 보고합니다.


## 판정 기준

| 판정 | 의미 |
|------|------|
| **PASS** | 원칙에 완전히 부합 |
| **WARN** | 부분적 우려 — 보완 조치로 해결 가능 |
| **FAIL** | 원칙에 위배 — 근본적 설계 변경 필요 |

## 예시

### 예시 1: PASS 사례

```
기능: "어노테이터에 AI 자동 분류 도구 추가"
- All-in-One: PASS — 기존 어노테이터 내 통합
- 라벨링 최적화: PASS — 작업 속도 직접 향상
- No/Low/High Code: PASS — UI에서 바로 사용 가능
- 보안: PASS — 기존 RBAC 내에서 동작
- 확장성: PASS — 플러그인으로 커스텀 모델 연동 가능
범위: In-Scope (플랫폼 내장)
```

### 예시 2: WARN 사례

```
기능: "DICOM 의료 영상 전용 뷰어"
- All-in-One: PASS — 어노테이터 확장
- 라벨링 최적화: PASS — 의료 도메인 효율성 향상
- No/Low/High Code: WARN — 의료 영상 전문 지식 필요할 수 있음
- 보안: WARN — 의료 데이터 HIPAA 등 추가 규정 확인 필요
- 확장성: PASS — 플러그인으로 분리 가능
범위: 플러그인 (SYNAPSE-MEDI) — 의료 도메인 특화
```

### 예시 3: FAIL 사례

```
기능: "웹 크롤링 기반 자동 데이터 수집"
- All-in-One: WARN — 데이터 파이프라인 확장이나 핵심은 아님
- 라벨링 최적화: FAIL — 라벨링과 무관한 기능
- No/Low/High Code: PASS — UI에서 URL 입력으로 사용 가능
- 보안: FAIL — 외부 데이터 수집은 보안 위험 증가
- 확장성: PASS — 플러그인으로 가능
범위: Out-of-Scope — 시냅스는 데이터 수집이 아닌 관리/라벨링 플랫폼
```
