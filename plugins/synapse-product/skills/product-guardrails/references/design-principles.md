# 시냅스 UX/제품 설계 원칙

## 1. 직관적 정보 계층 (Intuitive Information Hierarchy)

### 원칙

사용자가 원하는 정보에 최소한의 클릭으로 도달할 수 있어야 한다.

### 적용

```
워크스페이스 → 프로젝트 → 워크샵/데이터 컬렉션/ML 실험
                    └── 최대 3단계 내에 핵심 기능 접근 가능
```

- 카드형 목록에서 핵심 정보를 즉시 파악
- 드릴다운 구조로 세부 정보 접근
- 브레드크럼으로 현재 위치 인지

### 검증 질문

- 사용자가 원하는 기능까지 몇 번의 클릭이 필요한가? (3회 이내 권장)
- 현재 화면에서 사용자가 다음에 할 행동이 명확한가?
- 불필요한 중간 페이지가 있지 않은가?

## 2. 작업 맥락 유지 (Context Preservation)

### 원칙

사용자가 작업 중인 맥락을 잃지 않아야 한다.

### 적용

- 어노테이터에서 라벨링 중 페이지 이동 없이 모든 작업 완료
- 대시보드에서 프로젝트 전체 상태를 한 화면에서 파악
- 설정 변경 시 현재 작업에 즉시 반영

### 위반 사례

- 라벨링 중 클래스 추가를 위해 프로젝트 설정 페이지로 이동해야 하는 경우
- 검수 중 원본 데이터를 보기 위해 데이터 컬렉션으로 이동해야 하는 경우
- 익스포트 설정에서 데이터 통계를 확인하기 위해 대시보드로 이동해야 하는 경우

## 3. 역할 맞춤 인터페이스 (Role-Tailored Interface)

### 원칙

각 역할에 필요한 기능만 노출하고, 불필요한 복잡성을 숨긴다.

### 역할별 인터페이스 설계

| 역할 | 최소 인터페이스 | 확장 인터페이스 |
|------|----------------|----------------|
| 작업자 | 어노테이터 + 태스크 목록 | 가이드라인, 튜토리얼 |
| 검수자 | 어노테이터 (리뷰 모드) + 검수 목록 | 검수 통계, 피드백 |
| 관리자 | 대시보드 + 프로젝트 설정 | 멤버 관리, 통계 상세 |
| 개발자 | SDK 문서 + API 레퍼런스 | 플러그인 관리, 디버그 도구 |

## 4. 실패 안전 설계 (Fail-Safe Design)

### 원칙

사용자의 실수로 인한 데이터 손실을 방지하고, 복구 경로를 제공한다.

### 적용

- 라벨링 작업 자동 저장 (일정 주기)
- 삭제 전 확인 대화상자 + 되돌리기 옵션
- 데이터 컬렉션 설정 변경 시 영향 범위 미리 알림
- 대량 작업 (삭제, 재할당) 시 dry-run 옵션

### 필수 규칙

- 데이터 삭제는 항상 soft delete 또는 확인 단계
- 포맷 변환 시 원본 데이터는 보존
- 중요 설정 변경 시 영향 범위 표시

## 5. 성능 인식 설계 (Performance-Aware Design)

### 원칙

대량 데이터에서도 응답성을 유지하고, 장기 작업은 비동기로 처리한다.

### 적용

| 데이터 규모 | 처리 방식 |
|------------|----------|
| < 100건 | 즉시 로드, 동기 처리 |
| 100-10K건 | 페이지네이션, 가상 스크롤 |
| 10K+ 건 | 비동기 처리, 스크립트 제출, 진행 상태 표시 |
| 100K+ 건 | BaseExporter + Ray 클러스터, 청크 처리 |

### 금지 사항

- 프론트엔드에서 10K+ 데이터를 한 번에 로드
- API 페이지네이션으로 100K+ 데이터를 순차 조회
- 장기 작업을 동기 API 호출로 처리

## 6. 일관된 패턴 (Consistent Patterns)

### UI 패턴

| 패턴 | 사용 위치 | 설명 |
|------|----------|------|
| 카드형 목록 | 프로젝트, 워크샵, 컬렉션 목록 | 썸네일 + 핵심 정보 |
| 테이블형 목록 | 태스크, 데이터 유닛, 멤버 목록 | 상세 정보 + 정렬/필터 |
| 마법사 (Wizard) | 프로젝트/워크샵/컬렉션 생성 | 단계별 입력 안내 |
| 설정 패널 | 어노테이터, 컬렉션 설정 | 탭/섹션 기반 설정 |
| 대시보드 | 프로젝트, 워크샵 | 차트 + 통계 카드 |

### 인터랙션 패턴

| 패턴 | 설명 |
|------|------|
| 인라인 편집 | 테이블 내 직접 수정 |
| 드래그&드롭 | 파일 업로드, 순서 변경 |
| 키보드 단축키 | 어노테이터 도구 전환 |
| 컨텍스트 메뉴 | 우클릭 추가 옵션 |
| 토스트 알림 | 작업 완료/오류 피드백 |

## 7. 접근성 및 국제화

### 접근성

- 키보드 네비게이션 지원
- 적절한 색상 대비
- 스크린 리더 호환 라벨

### 국제화

- 한국어/영어 기본 지원
- UI 텍스트 외부화 (하드코딩 금지)
- 날짜/시간 형식 로케일 대응
- RTL 레이아웃 고려 (향후)
