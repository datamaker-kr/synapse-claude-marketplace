# 시냅스 기능 범위 경계

## 범위 판단 프레임워크

새 기능이 시냅스의 범위에 속하는지 판단하기 위한 의사결정 트리.

```
1. 이 기능이 AI 학습 데이터 구축/관리와 관련이 있는가?
   ├── YES → 2단계로
   └── NO → 범위 밖 (Out-of-Scope)

2. 기존 End-to-End 파이프라인의 어느 단계에 해당하는가?
   ├── 데이터 관리 → 3단계로
   ├── 어노테이션 → 3단계로
   ├── ML 파이프라인 → 3단계로
   ├── 프로젝트 운영 → 3단계로
   └── 해당 없음 → 범위 밖 or 플러그인 고려

3. 모든 사용자에게 필요한가, 특정 도메인에 한정되는가?
   ├── 모든 사용자 → 플랫폼 내장 기능
   └── 특정 도메인 → 플러그인으로 분리
```

## 모듈별 범위 상세

### 데이터 컬렉션 범위

#### In-Scope

- 다양한 데이터 유형 지원 (이미지, 비디오, 텍스트, 3D, 오디오, JSON)
- 파일 스펙 정의 및 관리
- 메타데이터 스키마 정의 및 유효성 검증
- 데이터 유닛 CRUD
- 데이터 업로드 (로컬, S3, GCS, SFTP)
- 데이터 익스포트 (JSON, COCO, YOLO, VOC, CSV)
- 데이터셋 버전 관리
- 데이터 분포/유사도 분석
- 플러그인을 통한 포맷 확장

#### Out-of-Scope

- 데이터 크롤링/스크래핑
- 데이터 합성/생성 (별도 플러그인 가능)
- 범용 파일 관리 시스템
- 데이터 정제/클리닝 도구 (기본 전처리만 지원)

### 어노테이터 범위

#### In-Scope

- 이미지/비디오/텍스트/3D/프롬프트 라벨링
- 어노테이션 타입: bbox, polygon, polyline, keypoint, classification, segmentation
- AI 보조 도구 (오토 라벨링, 자동 감지, 자동 추적)
- 전/후처리 자동화
- 자동 품질 검토
- 어노테이터 UI 커스터마이징 (툴바, 에디터, 패널)
- 키보드 단축키
- 플러그인 기반 도구 확장

#### Out-of-Scope

- 범용 이미지 편집기 (포토샵 대체가 아님)
- 3D 모델링 도구 (라벨링만 지원)
- 비디오 편집 (프레임 단위 라벨링만 지원)
- 텍스트 편집기/워드프로세서

### 프로젝트 관리 범위

#### In-Scope

- 프로젝트 CRUD 및 설정
- 워크샵 관리 (작업 할당, 검수, 진행 관리)
- 태스크 관리 및 할당
- 대시보드 (어노테이션 관련 통계)
- 튜토리얼 (작업자 교육/선별)
- 멤버 관리 및 RBAC

#### Out-of-Scope

- 범용 프로젝트 관리 (Jira, Asana 대체가 아님)
- 스프린트/애자일 보드
- 작업자 급여/인사 관리
- 고객 커뮤니케이션 (Slack, Teams 대체가 아님)
- 타임 트래킹 (작업자 시간 관리)

### ML 파이프라인 범위

#### In-Scope

- ML 실험 생성 및 관리
- 모델 학습 실행 (GPU 연동)
- 학습 모니터링 및 메트릭 비교
- 모델 라이브러리 관리
- 모델 배포 (플러그인, 추론 서버)
- Ground Truth 데이터셋 관리

#### Out-of-Scope

- 범용 ML 프레임워크 (TensorFlow, PyTorch 대체가 아님)
- AutoML 플랫폼 (기본 최적화만 지원)
- 독립 추론 서버 인프라 (배포 지원만)
- ML 모델 마켓플레이스

### 스토리지/인프라 범위

#### In-Scope

- 사용자 스토리지 연동 (S3, GCS, SFTP, Local)
- GPU 서버 연동
- 폐쇄망 환경 지원
- 데이터 보안 (격리, 암호화, RBAC)

#### Out-of-Scope

- 클라우드 인프라 관리 (AWS Console 대체가 아님)
- 스토리지 프로비저닝
- 네트워크 구성/관리
- 인프라 모니터링 (Grafana 대체가 아님)

## 경계 사례 판단 가이드

### 판단이 어려운 경우

| 기능 요청 | 판단 | 근거 |
|----------|------|------|
| "데이터 증강 기능 추가" | 플러그인 | 도메인별 증강 방식이 다름; 핵심 기능에 통합하면 복잡도 증가 |
| "어노테이션 가이드라인 에디터" | In-Scope | 프로젝트 운영의 핵심 요소; 작업 품질에 직접 영향 |
| "크라우드소싱 마켓플레이스" | Out-of-Scope | 작업자 관리는 지원하나, 외부 마켓플레이스 구축은 범위 밖 |
| "사전 학습 모델 파인튜닝" | 플러그인 | 모델별 파인튜닝 방식이 달라 플러그인이 적합 |
| "데이터 유사도 기반 중복 제거" | In-Scope | 데이터 품질 관리의 핵심 기능; 모든 사용자에게 유용 |
| "Slack 알림 연동" | 플러그인 | 특정 커뮤니케이션 도구에 한정; 플러그인이 적합 |
| "어노테이션 버전 관리" | In-Scope | 라벨링 품질 관리의 핵심; 되돌리기/비교에 필수 |
| "클래스 자동 추천" | In-Scope | AI 보조 기능의 일부; 라벨링 효율성 향상 |

## SYNAPSE+X 확장 모델

시냅스 코어 플랫폼을 기반으로 도메인 특화 솔루션을 제공하는 확장 모델.

```
SYNAPSE (Core)
├── SYNAPSE-MEDI (의료)
│   └── 의료 영상 분석, 진단 자동화, 치료 효과 예측
├── SYNAPSE-MFG (제조)
│   └── 결함 검출, 품질 검사
├── SYNAPSE-FIN (금융)
│   └── 문서 분석, 이상 거래 탐지
└── SYNAPSE-X (기타 도메인)
    └── 도메인 맞춤형 플러그인 + 워크플로우
```

### 확장 원칙

- Core 플랫폼의 기능은 모든 SYNAPSE+X에서 공유
- 도메인 특화 기능은 플러그인/설정으로 분리
- Core 변경이 모든 확장에 호환되어야 함 (하위 호환성 필수)
