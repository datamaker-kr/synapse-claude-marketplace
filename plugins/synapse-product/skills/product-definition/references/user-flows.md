# 시냅스 주요 사용자 흐름

## 1. 데이터 구축 흐름 (Data Pipeline)

```
[데이터 관리자]
1. 워크스페이스 진입
2. 프로젝트 생성 (제목, 설명, 설정)
3. 데이터 컬렉션 생성
   a. 파일 스펙 정의 (메인 파일셋 + 서브 파일셋)
   b. 메타데이터 스키마 정의 (선택)
   c. 콘텐츠 제한 / 공개 여부 설정
4. 데이터 업로드
   a. 소스 선택 (로컬, S3, GCS, SFTP)
   b. 파일-스펙 매핑 (자동 또는 수동)
   c. 메타데이터 첨부 (Excel 등)
   d. 데이터 유닛 생성
5. 데이터 확인 및 관리
```

## 2. 어노테이션 흐름 (Annotation Pipeline)

```
[프로젝트 관리자]
1. 워크샵 생성 (프로젝트 내)
   a. 작업 유형 선택
   b. 클래스 및 속성 정의
   c. 가이드라인 작성
2. 작업 할당 설정
   a. 작업자 배정 방식 설정
   b. 검수 규칙 설정
   c. 태스크 자동/수동 할당

[라벨링 작업자]
3. 태스크 수령 → 어노테이터 진입
4. 라벨링 수행
   a. 도구 선택 (bbox, polygon, keypoint 등)
   b. 클래스 할당
   c. 속성 입력
   d. AI 보조 활용 (오토 라벨링, 자동 감지)
5. 결과 저장 및 제출

[검수자]
6. 검수 수행
   a. 자동 검토 규칙 적용
   b. 수동 검수
   c. 반려 또는 승인
7. 반려 시 → 작업자에게 재작업 요청
```

## 3. 모델 학습 흐름 (ML Pipeline)

```
[ML 엔지니어]
1. ML 실험 생성
   a. 실험 목적 정의
   b. 데이터셋 선택 (Ground Truth)
2. 학습 실행
   a. 신경망 아키텍처 선택
   b. 하이퍼파라미터 설정
   c. 컴퓨팅 자원 할당 (GPU)
   d. 학습 실행
3. 학습 모니터링
   a. 실시간 메트릭 확인
   b. 여러 실험 동시 실행/비교
4. 모델 배포
   a. 최적 모델 선택
   b. 모델 라이브러리 등록
   c. 플러그인으로 재사용 또는 추론 서버 배포
```

## 4. 데이터 익스포트 흐름 (Export Pipeline)

```
[데이터 관리자 / ML 엔지니어]
1. 익스포트 대상 선택
   - Assignment: 라벨링 결과
   - Ground Truth: 큐레이션된 학습 데이터
   - Task: 태스크 메타데이터 + 어노테이션
2. 출력 포맷 선택
   - JSON (원본, 무손실)
   - COCO (객체 감지/세그멘테이션)
   - YOLO (darknet 형식)
   - Pascal VOC (XML)
   - CSV
   - 커스텀 포맷
3. 추가 옵션 설정
   - 원본 파일 포함 여부
   - Train/Val/Test 분할 비율
   - 필터 (상태, 태그, 날짜 범위)
   - 클래스 매핑
4. 익스포트 실행
5. 결과 다운로드/활용
```

## 5. 플러그인 확장 흐름 (Plugin Pipeline)

```
[플러그인 개발자]
1. synapse-sdk 설치
2. 플러그인 개발
   a. 액션 클래스 구현 (Train, Export, Upload, Inference, Deployment)
   b. config.yaml 작성
   c. 스텝 워크플로우 구성
3. 로컬 테스트
4. 플러그인 배포
   a. 플러그인 등록
   b. 워크스페이스에서 설치
5. 사용자가 플러그인 활용
   - 어노테이터 내 커스텀 도구로 활용
   - 전/후처리 자동화
   - 커스텀 익스포트/변환
```

## 6. 튜토리얼 흐름 (Tutorial Pipeline)

```
[프로젝트 관리자]
1. 튜토리얼 생성
   a. 교육 데이터 구성
   b. 정답 데이터 설정
   c. 채점 기준 정의
2. 작업자에게 튜토리얼 할당

[라벨링 작업자]
3. 튜토리얼 수행
   a. 교육 데이터로 연습
   b. 자동 채점으로 결과 확인
4. 합격 시 → 실제 프로젝트 작업 투입
```

## 역할별 페이지 접근 패턴

| 역할 | 주요 접근 페이지 |
|------|----------------|
| 워크스페이스 소유자 | 모든 페이지 + 멤버 관리 + 스토리지 + 플러그인 |
| 프로젝트 관리자 | 프로젝트, 워크샵, 대시보드, 데이터 컬렉션 |
| 데이터 관리자 | 데이터 컬렉션, 업로드, 익스포트 |
| 라벨링 작업자 | 어노테이터, 워크샵 태스크 |
| 검수자 | 어노테이터 (리뷰 모드), 워크샵 |
| ML 엔지니어 | ML 실험, 모델 라이브러리, 데이터셋 |
